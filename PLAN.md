## 1. 조사 요약 — Fleek은 어떤 앱인가

**Fleek – Workout Tracker, Log** (Android `com.primero.fleekfrontend` / iOS id1576993198)

- 서울 소재 DLT파트너스가 2021년 출시, 현재 MWM/Primero가 배포하는 헬스(웨이트) 기록 앱. 슬로건 "Logging Maketh Muscle".
- 다운로드 35만+, 평점 ~4.7. 무료 + Fleek Pro 구독(루틴 4개 제한, 일부 통계 차트 유료).
- **웹 버전 없음** — app.fleek.fit은 다운로드 랜딩 페이지뿐. 제품 전체가 모바일 앱.
- **치명적 약점(리뷰 검증됨)**: 서버 의존 — 오프라인 사용 불가, 로그인 오류, 2025년 9월 서버 사고로 다년치 기록 소실, 휴식 타이머는 포그라운드에서만 동작, 완료된 기록 수정 불가(삭제 후 재입력), 커스텀 운동은 통계에서 제외.
  → **로컬 오프라인 재구축이 이 약점들을 정면으로 해결한다.**

### 앱 구조 (공식 스크린샷 8장 기반 재구성)

| 화면 | 작동 방식 |
|---|---|
| **Record 탭** | 월 캘린더 + 날짜별 점 표시(운동/사진/신체), 선택일의 세션 리스트 → 상세 |
| **Routine** | 루틴 상세: 전/후면 해부도에 타깃 근육 셰이딩 + 운동 리스트(썸네일, 별칭, 근육 태그) |
| **로깅 화면** | 상단: 경과 타이머 + 진행 점. 본문: 운동 일러스트, 메모, 세트 테이블 **Set \| KG \| Reps \| 완료✓** (이전 세션 값이 미리 채워짐), 세트 타입(웜업/드롭/실패), +세트/−세트/모두완료/다음 운동. 세트 완료 시 휴식 타이머 자동 시작(기본 60초, 알람) |
| **운동 요약** | 완료 시: 전/후면 근육 그림 + 통계 그리드(회차 "331st", 칼로리, 시간, 볼륨 kg, 운동/세트/렙 수, 강도 kg/min) + 운동별 MAX 무게·1RM·세트 칩(20×12, 30×10…) |
| **운동 상세** | Instructions(팁/주의) + My Records: **Top Records(1RM MAX·VOLUME MAX·WEIGHT MAX)** / Recent Records / Chart |
| **Stats 탭** | **근육 피로도**: 전/후면 해부도 히트맵 + 과거로 스크럽하는 시간 슬라이더 / **Used Muscle**: 7·30·90일·전체 기간별 사용 근육 통계 |
| 소셜/AI/트레이너/워치 | 서버 의존 기능 — 재구축 제외 |

### 운동 라이브러리
400+ 종목, 부위·기구별 분류, 종목별 애니메이션·근육 다이어그램·별칭·팁. 커스텀 운동 추가 가능(단, Fleek에서는 통계 미지원 — 최다 불만).

## 2. 재구축 스코프

### 포함 (핵심 기능)
1. **운동 라이브러리** — 시드 57종목(주동근/협응근 매핑) + 커스텀 운동(내장 운동과 동급으로 통계 지원)
2. **루틴 빌더** — 이름 + 운동 순서 + 세트별 목표(무게·횟수) + 운동별 휴식 시간
3. **워크아웃 로깅** — 빈 운동 또는 루틴에서 시작, 세트 테이블(이전 기록 미리 채움), 세트 타입, 세트 완료 체크 → 휴식 타이머 자동 시작(+15/−15/건너뛰기), 진행 중 세션은 localStorage에 상시 보존(앱 종료에도 생존)
4. **운동 요약** — 회차·시간·볼륨·칼로리(MET 추정)·운동/세트/렙·강도(kg/min) + 세션 근육 히트맵 + 운동별 브레이크다운 + PR 배지
5. **기록** — 월 캘린더(운동일 점 표시) + 리스트, 상세 보기, **사후 수정·삭제·반복**(Fleek 개선점)
6. **운동별 기록·차트** — Top Records(1RM MAX/VOLUME MAX/WEIGHT MAX), 최근 기록, e1RM·볼륨 추이 라인 차트
7. **근육 피로도 해부도** (시그니처) — 전/후면 SVG 히트맵 + 시간 슬라이더, Used Muscle 7/30/90일/전체
8. **설정** — 체중(칼로리 계산용), 기본 휴식 시간, 데이터 JSON 내보내기/가져오기

### 제외 (서버·플랫폼 의존)
소셜(팔로우·Top Percentage 백분위), 트레이너 모드, AI 루틴 추천, InBody·헬스장 기구 연동, 워치·HealthKit, 진행 사진, 계정·구독·광고

### Fleek 대비 의도적 개선
- **완전 오프라인** (데이터 소실·로그인 문제 원천 차단)
- 완료된 워크아웃 **사후 수정 가능**
- 커스텀 운동도 **통계·차트 동급 지원**
- 진행 중 세션 **자동 보존**

## 3. 아키텍처

- **스택**: Vite + React 18 + TypeScript(strict) — PWA(매니페스트+서비스워커)로 PC·안드로이드 어디서든 설치형 로컬 앱
- **저장**: localStorage 단일 JSON 문서(`AppData v1`) — 단일 사용자 로컬 앱에 충분(Flexify/Massive 검증 패턴), 모든 통계는 세트 로그에서 **파생 계산**(저장 안 함)
- **차트**: 외부 의존성 없는 자체 SVG(라인·바), 해부도는 자체 BodyMap 컴포넌트

### 데이터 모델 (Hevy OpenAPI 스키마 벤치마크)
```ts
Exercise { id, name(한), nameEn, equipment, primaryMuscles[], secondaryMuscles[], isCustom? }
Routine { id, name, exercises: [{ exerciseId, restSec?, sets: [{ type, targetWeight?, targetReps? }] }] }
Workout { id, name, startedAt, endedAt?, exercises: [{ exerciseId, sets: [{ weight, reps, type, done, doneAt? }], memo? }] }
Settings { defaultRestSec, bodyWeightKg }
MuscleId = 17개 부위 (가슴·전/측/후면어깨·이두·삼두·전완·복근·복사근·대퇴사두·햄스트링·둔근·종아리·광배·승모·등상부·허리)
```
루틴 세트와 로깅 세트가 동일한 Set 형태 공유(루틴 = 목표값이 있는 워크아웃 템플릿). 이전 세션 값 = 같은 exerciseId의 마지막 완료 세트.

### 근육 피로도 알고리즘 (Fitbod/Hevy 관례 기반 재구성*)
```
세트 임펄스 I = 역할가중치(주동근 1.0 / 협응근 0.5) × K (K≈16.7 → 하드세트 6개 ≈ 100%)
피로도(m, t) = min(100, Σ 최근 7일 세트: I × exp(−Δt/τ))
  τ = 대근육(대퇴사두·햄스트링·둔근·광배·가슴·허리) 34h / 소근육 22h
7일 이상 미훈련 근육은 0으로 강제 리셋. 회복률 = 100 − 피로도.
색상: 피로 낮음(기본색) → 높음(네온 라임 → 주황/빨강 톤 보간)
Used Muscle(기간 W) = Σ 분수 세트(1.0/0.5), 최대값 대비 상대 히트맵 + 순위 리스트
칼로리 ≈ MET 5.0 × 체중(kg) × 시간(h)
e1RM = Epley: w×(1+r/30), 웜업 제외
```
\* Fleek은 공식을 비공개 — 검증된 것은 "세트 로그에서 계산, 며칠에 걸쳐 감쇠, 시간 슬라이더, 7/30/90일 집계"뿐이며 위 수식은 업계 표준 재구성(자유 튜닝 가능).

### 화면 구성 (하단 5탭)
```
홈    — 주간 요약, 빈 워크아웃 시작, 루틴 퀵스타트, 진행 중 세션 이어가기, 설정
루틴  — 루틴 목록/생성/편집(운동 선택 시트), 루틴에서 시작
기록  — 월 캘린더 + 점, 선택일 세션 리스트 → 상세(수정/삭제/반복)
통계  — 근육 피로도 해부도 + 시간 슬라이더, Used Muscle 7/30/90/전체, 주간 볼륨 바 차트
운동  — 라이브러리 검색/부위 필터, 상세(근육도 + Top Records + 차트), 커스텀 추가
+ 전역: 활성 워크아웃 화면(로깅), 휴식 타이머 바, 요약 시트
```

## 4. 개발 단계

| 단계 | 내용 | 완료 기준 |
|---|---|---|
| P0 | 스캐폴드: Vite+React+TS, 타입, 공식, 저장 계층, 시드 운동, 다크 테마, BodyMap SVG | `tsc` 통과 ✅(완료) |
| P1 | 앱 셸: 5탭 내비게이션 + 전역 상태 저장소(useReducer + localStorage 동기화) | 탭 전환 동작 |
| P2 | 코어 루프: 라이브러리 → 루틴 빌더 → 워크아웃 로깅(세트 체크·이전값 채움·휴식 타이머) → 요약 | 루틴 생성→기록→요약 E2E |
| P3 | 기록: 캘린더 + 상세 + 수정/삭제/반복 | 과거 기록 편집 가능 |
| P4 | 통계: 피로도 해부도 + 슬라이더, Used Muscle, 운동별 Top Records·차트, 주간 볼륨 | 기록 입력 시 통계 반영 |
| P5 | 검증: 타입체크·빌드, 프리뷰 브라우저 E2E 클릭스루, 멀티에이전트 코드 리뷰, 버그 수정 | 전 플로우 무결함 |

## 5. 테스트 계획
1. `tsc -b` + `vite build` 클린 통과
2. 프리뷰 서버 E2E 시나리오: 루틴 생성(운동 3개) → 루틴으로 워크아웃 시작 → 세트 입력·완료(타이머 확인) → 종료·요약 확인 → 기록 캘린더에서 확인 → 수정 → 통계(피로도·Used Muscle·차트) 반영 확인 → 커스텀 운동 추가 후 동일 플로우 → 새로고침 후 데이터 유지 확인
3. 멀티에이전트 적대적 코드 리뷰(정확성·데이터 무결성·엣지 케이스) 후 수정
