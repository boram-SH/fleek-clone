IDENTITY & CONTEXT. "Fleek - Workout Tracker, Log" (Android package com.primero.fleekfrontend; iOS id1576993198) is a freemium gym-logging app by DLT Partners Inc. (Seoul, Korea; contact busking@fleek.fitness), distributed under the MWM/Primero publisher umbrella. Tagline: "Logging Maketh Muscle." ~4.8 rating, 352K+ downloads; Fleek Pro at $2.99/mo, $23.99/yr, $69.99 lifetime. Latest Android version 3.7.34 (June 2026).

SCREENS (from MWM's screen-by-screen UX captions). (1) Exercise Guide: "Push & Pull" exercise categories plus detailed anatomical muscle diagrams per exercise; 400+ exercises with GIFs, plus user-defined custom exercises. (2) Workout Logging: per-set "KG" and "Reps" inputs with a real-time "Rest Time" counter between sets. (3) Performance Review: "Top Records" (1RM MAX) and "Recent Records" history per exercise. (4) Exercise Tips: form "Tips" and "Precautions" text per exercise. (5) Performance Benchmarking: "Top Percentage" — your 1RM percentile vs other users (server-backed social feature). (6) Workout Summary: post-session totals — "Calories" burned and total "Volume" lifted across all exercises. (7) Muscle Fatigue: the signature screen — current muscle fatigue rendered as a heat map on anatomical body figures (front/back "3D body model"), plus "Used Muscle" statistics selectable over 7 / 30 / 90 days / total. (8) Workout History: calendar interface + chronological workout list. Also: dynamic progress charts, routine creation/upload, following other users' logs, Apple Watch sync.

FATIGUE/RECOVERY MECHANICS. Fleek does not publish its formula, but its model matches the industry-standard approach best documented by Fitbod: each muscle group gets a 0–100% recovery (inverse fatigue) score computed only from logged training history ("muscle-group-local," no HRV/wearable signals needed). Fitbod's documented behavior: every logged exercise's sets/reps/weight are summed per targeted muscle group; typical muscle recovery takes ~2–3 days (48–72h); the algorithm force-resets any muscle to 100% recovered after 7 days of rest; the Recovery tab shows a composite body heat map color-coded by percentage, with a manual "Fresh Muscle Groups" override. Per-muscle attribution convention (used by Hevy and standard hypertrophy-volume tools): each exercise declares primary and secondary muscles; a set counts 1.0 toward each primary muscle and 0.5 (fractional set) toward each secondary muscle. Weekly landmark: 10–20 hard sets per muscle per week (RP volume landmarks), useful as the normalization constant.

CONCRETE LOCAL ALGORITHM (implementable offline). Data model: Exercise{id, name, category, equipment, primaryMuscles[], secondaryMuscles[]}; Workout{id, startedAt, endedAt}; SetLog{workoutId, exerciseId, weightKg, reps, completedAt}. Muscle taxonomy ~15–20 groups covering both body views: chest, front/side/rear delts (or one "shoulders"), biceps, triceps, forearms, traps, lats, upper back, lower back, abs, obliques, glutes, quads, hamstrings, calves, neck(optional). Seed data: yuhonas/free-exercise-db (public domain, 800+ exercises, JSON with primaryMuscles/secondaryMuscles fields + images) or exercemus/exercises (merged wger + exercises.json).

Step 1 — per-set fatigue impulse. For each logged set s of exercise e: roleWeight r = 1.0 (primary) / 0.5 (secondary). Optional intensity scaling: e1RM via Epley = weight×(1+reps/30); intensityFactor = clamp(weight×reps / (e1RM×8), 0.5, 1.5) — treat ~8 reps at working weight as a canonical "hard set." Impulse I(s,m) = r × intensityFactor (or simply r for v1).

Step 2 — accumulation with decay. currentFatigue(m, now) = min(100, Σ over sets in the last 7 days: I(s,m) × K × exp(-Δt/τ(m))), where Δt = hours since the set, K ≈ 100/6 (≈16.7 fatigue points per hard set, so ~6 hard sets in one session ≈ 100% fatigued), and τ(m) = time constant: large muscles (quads, hamstrings, glutes, lats, chest, lower back) τ ≈ 30–36h (≈90% recovered by 72h); small muscles (biceps, triceps, delts, calves, abs, forearms, traps) τ ≈ 18–24h (≈90% recovered by 48h). Hard-clamp fatigue to 0 for any muscle untrained ≥7 days (Fitbod convention). recovery% = 100 − fatigue%. Alternative linear variant (simpler, what many clones use): fatigue(t) = fatigue0 × max(0, 1 − Δt/T(m)), T = 48h small / 72h large muscles. Recompute lazily on screen open — pure function of the set log, no background jobs.

Step 3 — body-map rendering. Render front + back SVG body silhouettes with one path/polygon group per muscle; fill color from recovery%: ≥80% neutral/green ("fresh"), 50–79% yellow/orange, <50% red ("fatigued"), optionally with opacity ∝ fatigue. Tap a muscle → detail sheet: recovery %, last-trained date, contributing exercises/sets, ETA to full recovery (solve exp decay for fatigue<20). Ready-made open-source SVG assets: react-native-body-highlighter (Expo-compatible, front/back models, per-muscle slugs, intensity levels), react-body-highlighter (web React port of the same polygons), lahaxearnaud/body-highlighter (framework-agnostic, auto-normalizes anatomical names like "Pectoralis Major" → muscle group), vulovix body-muscles (zero-dependency, 70+ muscles).

Step 4 — "Used Muscles" 7/30/90/total stats. For window W: usage(m) = Σ fractional sets (1.0/0.5) over W, and/or tonnage(m) = Σ weight×reps×r. Display: (a) same body figure with intensity = usage(m)/max usage (relative heat map — this is Fleek's "Used Muscle" view), (b) ranked bar list per muscle with set counts, (c) optional sets-per-week vs the 10–20 target band (Hevy-style, with weekly/monthly grouping and muscle filter). Total-volume, duration, and calories for the summary screen: kcal ≈ MET(3.5–6 for resistance training) × bodyweightKg × durationHours.

REBUILD SCOPE NOTES. Core loop = pick exercise → log kg/reps sets with rest timer → summary (volume/calories) → history calendar → fatigue body map + used-muscle stats + per-exercise records/charts (e1RM via Epley). Server-dependent features to drop for a local app: "Top Percentage" social benchmarking, following users, routine upload/sharing, AI plan generation, Apple Watch sync. Everything in the fatigue/stats engine above is a pure function over the local set log, so it works fully offline.

FEATURES:
- Workout logging (sets: kg × reps + rest timer) [core=true]: Start a workout, add exercises, log each set with weight (KG) and reps inputs; a real-time rest-time counter runs between sets. Sets are checked off as completed; previous session's numbers shown as placeholders (standard pattern). Data written to local SetLog rows (workoutId, exerciseId, weightKg, reps, completedAt).
- Exercise library with muscle mapping [core=true]: 400+ exercises organized by category (e.g., Push & Pull) and equipment, each with an anatomical muscle diagram, form Tips and Precautions text, and demo GIF/image. Each exercise declares primaryMuscles[] and secondaryMuscles[] — this mapping is the input to the fatigue and used-muscle engines. Users can add custom exercises (name + muscle assignment). Seedable from the public-domain free-exercise-db JSON (800+ exercises with primary/secondary muscle fields).
- Muscle fatigue body map (signature feature) [core=true]: Front/back anatomical body figure where each muscle region is color-coded by current fatigue (0–100%). Computed purely from logged history: each set adds a fatigue impulse to its exercise's primary muscles (weight 1.0) and secondary muscles (0.5), scaled so ~6 hard sets ≈ 100% fatigue; fatigue decays exponentially with per-muscle time constants (small muscles ~recover in 48h, large in ~72h; hard reset to fully recovered after 7 days idle). Recovery% = 100 − fatigue%. Color bands: ≥80% fresh, 50–79% moderate, <50% fatigued/red. Tapping a muscle shows recovery %, last trained date, contributing sets, and ETA to recovery. Render with an SVG body-highlighter component (open-source polygon sets exist).
- Used Muscles statistics (7/30/90 days/total) [core=true]: Same body figure reused as a usage heat map over a selectable window (7/30/90 days, all time): per-muscle intensity = fractional sets (1.0 primary / 0.5 secondary) or tonnage summed in the window, normalized to the max muscle. Accompanied by a ranked per-muscle list (set counts/volume) and optionally weekly sets-per-muscle vs the 10–20 set target band.
- Workout summary screen [core=true]: Shown after finishing a session: total Volume lifted (Σ weight×reps across all sets), total Calories burned (MET-based estimate: MET 3.5–6 × bodyweightKg × hours), duration, exercise/set counts, and a muscle heat map of what the session hit.
- Workout history (calendar + list) [core=true]: Calendar view with workout-day markers plus a chronological list of past sessions; tapping opens the full logged workout (exercises, sets, per-session totals).
- Records & progress charts [core=true]: Per-exercise Performance Review: Top Records (1RM MAX, best weight/reps) and Recent Records history, plus dynamic line charts of progress over time. Estimated 1RM via Epley formula weight×(1+reps/30), recomputed from the set log.
- Routines/programs [core=true]: Create and save named routines (ordered exercise templates with target sets/reps); start a workout from a routine so logging is pre-filled. Fleek also supports uploading/sharing routines to other users (server feature — drop sharing, keep local create/reuse).
- Social benchmarking (Top Percentage) [core=false]: Compares the user's 1RM against other users' data as a percentile ('Top X%'), and lets users follow others and view their workout records. Requires a backend + user population — exclude from the local offline rebuild.
- AI plan personalization [core=false]: Marketing-level AI feature that adapts workout plans to performance/goals; no concrete published mechanics. Not needed for the core loop; a fresh-muscle-first exercise suggester (pick muscles with recovery ≥80%) reproduces the useful part locally.
- Apple Watch / wearable sync [core=false]: Log workouts from the watch and sync health data. Platform-specific integration, not part of the core local rebuild.
