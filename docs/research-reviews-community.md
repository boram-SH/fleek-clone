SUMMARY:
Fleek - Workout Tracker, Log (package com.primero.fleekfrontend) is a gym-focused weightlifting logger originally built by Korean developer DLT Partners Inc. (Korean name "플릭"), now distributed under the MWM app portfolio. Rated ~4.8 with 350K+ downloads; free with a Pro subscription ($2.99/mo, $23.99/yr, $69.99 lifetime; KRW 3,900/29,000/89,000 in Korea). Because the app is Korean-origin, the richest real-user commentary is in Korean communities (Clien, Blind, theqoo) rather than Reddit, where it has almost no presence.

DAY-TO-DAY USAGE (what users actually do): The core loop reported by users is: open the app -> start a workout from a saved routine (or ad-hoc) -> pick exercises from a ~400-exercise library organized by body part and push/pull category -> log each set as weight (KG) + reps (time for cardo/timed work) -> a rest timer runs between sets (with recommended rest durations) -> finish the workout and get a summary (total volume lifted, calories, duration). Records land in a calendar/history view and feed charts. Users repeatedly describe it as the app for people "who train with the same routine every time" — the saved-routine + quick-logging flow is the product's center of gravity. Exercise entries show demonstration GIFs and anatomical muscle diagrams (which muscles an exercise hits), and users can attach notes per exercise and create custom exercises.

WHAT USERS PRAISE: (1) Simple, uncluttered logging UI — "the UI is pretty simple and doesn't overcomplicate things"; App Store reviewers call it an app "without inconveniences." (2) Well-organized stats: analysis by total volume, 1RM (estimated one-rep max with a "Top Percentage" benchmark comparing your 1RM against other users), and per-body-part breakdowns, which reviewers say makes goal setting easy. (3) The muscle fatigue visualization: an anatomical body figure showing current muscle fatigue per muscle group based on recent logged work, plus "Used Muscle" statistics over 7/30/90 days/total — users say they use it to decide what to train today and to verify a workout actually hit the intended muscle. (4) The AI routine recommendation with gym-equipment registration: you register which gym you attend / which machines it has, and the app generates routines using only available equipment, with specific weights and form notes; Clien users called this "a completely different level" because it answers "what should I do, how, and how much." InBody (body composition) integration further personalizes recommendations. (5) Exercise GIFs + form guidance, valued by beginners. (6) Non-aggressive monetization (historically) — free tier is fully usable for logging; (7) Apple Watch app (log sets, heart rate, rest timer from wrist; HealthKit sync) — note this is iOS-only, so not relevant for an Android/local rebuild core.

WHAT USERS COMPLAIN ABOUT: (1) Cardio logging is weak — a Blind thread shows a beginner unable to sensibly log treadmill/stepmill work when the machine shows no km; commenters said Fleek is great for resistance training but they track cardio elsewhere (Apple Watch). (2) In 2025 ads were introduced even for paying annual subscribers with no notice, causing churn threats on Clien (a sign of post-acquisition monetization pressure — a local offline rebuild trivially avoids this). (3) Account/login problems: Apple-login sync errors, losing access after changing phones, no easy account recovery — arguing for local-first data storage. (4) Free-tier friction: exercise-substitution suggestions and some AI features are paywalled; a "one week free" trial confused users; the lifetime plan was discontinued at one point. (5) Newer competitors (Raise/Rise, Planfit, Hevy) have pulled some users away.

ESSENTIAL vs GIMMICK (per user sentiment): Essential = set logging (weight x reps with previous-session values visible), saved routines, rest timer, calendar history, progress charts (volume/1RM per exercise), body-part exercise library with custom exercises, and the muscle fatigue/used-muscle anatomy view (Fleek's signature differentiator, frequently cited). Secondary/gimmicky for a personal offline app = social feed (follow users, copy their routines, cheer/compete), celebrity/influencer routine library, trainer-trainee mode (trainers dispatch routines and review member logs — a real feature but for a different persona), AI adaptive plans and Top-Percentage benchmarking (needs server data), Apple Watch/HealthKit integration, and expert tips content (form/nutrition/recovery articles). PR (personal record) auto-notifications are a small but liked touch.

REBUILD IMPLICATIONS: The offline core should replicate: routine templates -> active workout session screen with per-set KG/reps entry and auto rest timer -> workout summary (volume/duration) -> calendar history -> per-exercise progress charts (est. 1RM, volume) -> muscle-group fatigue heatmap computed from logged sets over rolling 7/30/90-day windows, rendered on a front/back anatomical figure. Exercise library needs body-part + push/pull categorization, target-muscle mapping per exercise (this mapping powers the fatigue view), and custom exercise creation. Cardio support should at least allow time-only entries (distance optional) to address the known pain point.

FEATURES:
- Workout session logging (sets: weight x reps) [core=true]: Start a workout (from a saved routine or ad-hoc), add exercises, and log each set with KG and reps inputs (time for cardio/timed exercises). Previous performance is visible so users can progressively overload. Session ends with a summary screen showing total volume lifted, calories, and duration. This is the daily-use core loop.
- Rest timer between sets [core=true]: Timer starts between sets during an active workout and counts rest time in the logging interface; the app suggests recommended rest durations. Frequently mentioned as part of the standard flow. Available on Apple Watch too (wrist logging + heart rate) but the watch part is iOS-only.
- Exercise library (~400 exercises) + custom exercises [core=true]: Exercises organized by body part and push/pull category, each with demonstration GIF, form notes, and an anatomical diagram of targeted muscles. Users can create their own custom exercises. The per-exercise target-muscle mapping is what powers the muscle fatigue feature.
- Routines (create, save, reuse) [core=true]: Users build named workout routines (exercise list with target sets/reps/weights), save them, and launch workouts from them. Users describe Fleek as ideal for people who repeat the same routines. Routines can also be imported/copied from followed users or trainers.
- Muscle fatigue visualization (signature feature) [core=true]: An anatomical front/back body figure displays current fatigue level per muscle group, computed from recently logged sets; plus 'Used Muscle' statistics over 7/30/90 days and all-time windows. Users use it to decide what to train today and to confirm a workout stimulated the intended muscles. The most-cited differentiator vs other loggers.
- History calendar + workout log list [core=true]: Calendar interface marking workout days, with chronological list of past sessions; tapping shows the full log of that day's exercises/sets. Notes can be attached to individual exercises.
- Progress statistics and charts [core=true]: Dynamic growth charts per exercise and overall: total volume, estimated 1RM tracking, per-body-part analysis. Users praised that records are analyzed 'by total weight, 1RM, and body part' making goal setting easy. Personal-record (PR) notifications fire automatically when a best is beaten.
- Cardio logging [core=false]: Supports cardio session entries (duration, optionally distance/speed). Known weak point: users complained machines like stepmills show no km so logging is awkward — a rebuild should allow time-only cardio entries. Many users track cardio outside the app.
- AI routine recommendation (gym equipment + InBody aware) [core=false]: User registers their gym (or its machine list) and the app generates personalized routines using only available equipment, with specific weight prescriptions and form guidance; InBody body-composition sync further tailors recommendations; plans adapt as performance data accumulates. Highly praised by Korean users but server/AI-dependent and partly paywalled — not needed for an offline core, though a static routine-template generator could approximate it.
- Social features (follow, share, copy routines) [core=false]: Follow other users, view their workout records in a feed, upload your routines, copy/download others' routines, cheer/compete. Celebrity/influencer routine library curated in-app. Gimmick territory for a personal offline rebuild.
- Trainer mode [core=false]: Personal trainers manage multiple trainees: create and dispatch routine prescriptions, review members' logs, give feedback. A distinct B2B-ish persona feature; not part of the personal core loop.
- Apple Watch + HealthKit integration [core=false]: Watch app logs sets, shows heart rate, runs rest timer from the wrist; writes exercise time to Apple Health. iOS-only; praised by watch owners but irrelevant to an offline Android-style rebuild.
- Top Percentage benchmarking [core=false]: Compares the user's estimated 1RM against other users' data to show a percentile ranking. Requires server-side population data; a nice-to-have, not core.
- Fleek Pro subscription gating [core=false]: Free tier covers full logging/records; Pro ($2.99/mo, $23.99/yr, $69.99 lifetime) unlocks advanced analytics, AI personalization, exercise substitutions. In 2025 ads were added even for paid users, generating backlash — an offline rebuild has no equivalent, but this maps to which features users considered non-essential.
